## Первый запуск

1. **Подключите** устройство к компьютеру через USB
2. Windows распознает его как **"Yubikey 5A"** (VID:PID 1050:0407)
3. Устройство готово к работе сразу — дополнительные драйверы не требуются

### Статусы светодиода:

| Режим | Поведение | Значение |
|-------|-----------|----------|
| **Ожидание** | Мигает 1 раз/сек | Устройство спит, ждёт команду |
| **Активен** | Мигает 4 раза/сек | Готово к работе |
| **Подтверждение** | Горит постоянно, мигает каждую секунду | Нажмите кнопку для подтверждения |
| **Обработка** | Мигает 20 раз/сек | Идёт вычисление, подождите |

## Установка PIN

PIN защищает ваш ключ от несанкционированного использования. Минимальная длина — **1 символ**.

### Через Windows Hello:

1. **Настройки** → **Аккаунты** → **Варианты входа**
2. Найдите **"Безопасный ключ"** → нажмите **"Добавить"**
3. Следуйте инструкциям — Windows предложит задать PIN

### Через Yubico Authenticator:

1. Установите [Yubico Authenticator](https://www.yubico.com/products/authenticator/)
2. Подключите ключ
3. В настройках выберите **"Изменить PIN"**

### Через TUI (复工复用):

```bash
cd pico-key-tui
python -m pico_key_tui
```

Выберите **"Установить/Изменить PIN"** в меню.

## Регистрация ключа в Windows

### Добавление ключа:

1. Откройте **Настройки** → **Аккаунты** → **Варианты входа**
2. Нажмите **"Добавить"** → **"Безопасный ключ"**
3. Вставьте ключ в USB
4. **Нажмите кнопку** на устройстве когда загорится светодиод
5. Придумайте имя для ключа
6. Готово! Теперь вы можете входить по ключу

### Вход в аккаунт:

1. На странице входа выберите **"Вход через безопасный ключ"**
2. Вставьте ключ
3. **Нажмите кнопку** на устройстве
4. Готово!

### Регистрация на сайтах:

1. На сайте нажмите **"Включить двухфакторную аутентификацию"**
2. Выберите **"Безопасный ключ"** / **"Security Key"**
3. Вставьте ключ и **нажмите кнопку**
4. Сохраните резервные коды

## Вход по отпечатку/лицу

Windows Hello поддерживает вход через биометрию **только с встроенным** сканером. Для внешнего ключа (как наш) вход работает через **PIN + кнопка**:

1. При входе Windows запросит PIN
2. Вставьте ключ
3. **Нажмите кнопку** на устройстве

> **Примечание:** Полноценный вход "одним касанием" (без PIN) возможен только если настроена **биометрия встроенным сканером** И ключ registered как passkey.

## Работа с OATH/TOTP

Устройство может хранить одноразовые пароли (TOTP/HOTP) как Google Authenticator, но аппаратно.

### Добавление аккаунта:

#### Через Yubico Authenticator:

1. Установите [Yubico Authenticator](https://www.yubico.com/products/authenticator/)
2. Подключите ключ
3. Нажмите **"+"** → **"Введите вручную"**
4. Введите:
   - **Имя:** `MyService`
   - **Ключ:** (секрет из настроек сервиса)
   - **Тип:** `TOTP` (или `HOTP`)
   - **Период:** `30` секунд (стандарт)
5. Нажмите **"Добавить"**
6. **Нажмите кнопку** на ключе для подтверждения

#### Через TUI:

```bash
python -m pico_key_tui
```

Выберите **"OATH"** → **"Добавить аккаунт"**.

### Получение кода:

1. Откройте **Yubico Authenticator**
2. Выберите аккаунт
3. **Нажмите кнопку** на ключе
4. Код отобразится в приложении

### Через клавиатуру (otp):

Устройство может **набирать код напрямую** как клавиатуру:

1. Настройте HOTP в OATH
2. На сайте в поле ввода пароля **нажмите кнопку** на ключе
3. Код будет введён автоматически

## Backup с 24 словами

**КРИТИЧЕСКИ ВАЖНО:** Если вы потеряете ключ без backup, все ваши аккаунты будут потеряны навсегда!

### Создание backup:

1. Откройте **Pico Key TUI**:
   ```bash
   python -m pico_key_tui
   ```
2. Выберите **"Backup"** → **"Создать backup"**
3. **Нажмите кнопку** на ключе для подтверждения
4. На экране появятся **24 слова** (например: `abandon ability able about above absent absorb abstract absurd abuse access accident account accuse...`)
5. **Запишите эти слова на бумагу!**
6. Храните бумагу в **безопасном месте** (сейф, банковская ячейка)
7. **НИКОГДА** не храните слова в электронном виде!

### Восстановление из backup:

1. Если ключ сломан/потерян — купите новый ESP32-S3 модуль
2. Прошейте прошивку (см. раздел "Прошивка")
3. Откройте **Pico Key TUI**
4. Выберите **"Backup"** → **"Восстановить из backup"**
5. Введите **24 слова** через пробел
6. **Нажмите кнопку** на ключе
7. Все ваши аккаунты и ключи будут восстановлены

### Правила безопасности:

| ✅ Делайте | ❌ Не делайте |
|-----------|---------------|
| Записывайте слова на бумаге | Не сохраняйте в телефоне/компьютере |
| Храните в сейфу | Не фотографируйте слова |
| Делайте несколько копий | Не отправляйте по email/мессенджерам |
| Используйте водонепроницаемую бумагу | Не делайте backup на USB-накопителе |

## Кастомизация LED

Вы можете настроить яркость, цвет и режим светодиода.

### Через TUI:

1. Запустите TUI:
   ```bash
   python -m pico_key_tui
   ```
2. Выберите **"Настройки"** → **"LED"**
3. Настройте параметры:
   - **Яркость:** 0-15 (0=выкл, 15=максимум)
   - **Драйвер:** WS2812, Neopixel, Pico LED, Pimoroni
   - **Порядок цветов:** RGB, GRB, RBG и т.д.
   - **GPIO пин:** номер пина для LED

### Через Vendor команды (для разработчиков):

```python
# Яркость (0-15)
vendor_command(0x09, {
    0x01: 0x0D,  # CTAP_CONFIG_PHY_LED_BRIGHTNESS
    0x02: {0x01: bytes([10])}  # яркость = 10
})

# Драйвер (1=Pico, 2=Pimoroni, 3=WS2812, 5=Neopixel)
vendor_command(0x09, {
    0x01: 0x0A,  # CTAP_CONFIG_PHY_LED_DRIVER
    0x02: {0x01: bytes([3])}  # WS2812
})
```

## Смена VID/PID и имени устройства

Вы можете изменить USB-идентификаторы и имя устройства.

### Смена имени:

1. Откройте файл `pico-keys-sdk/src/usb/usb_descriptors.c`
2. Найдите строку:
   ```c
   "Yubikey 5A",                     // 2: Product
   ```
3. Замените на желаемое имя:
   ```c
   "My Custom Key",                  // 2: Product
   ```
4. Пересоберите и прошейте

### Смена VID/PID:

1. Откройте `sdkconfig`
2. Измените:
   ```
   CONFIG_USB_VID=0x1050
   CONFIG_USB_PID=0x0407
   ```
3. Пересоберите и прошейте

### Через Vendor команду (без перепрошивки):

```python
# Сменить VID:PID на 1234:5678
vendor_command(0x09, {
    0x01: 0x08,  # CTAP_CONFIG_PHY_VIDPID
    0x02: {0x01: bytes([0x12, 0x34, 0x56, 0x78])}
})
```

> **Важно:** Используйте только VID/PID, которыми вы владеете. Для тестов можно использовать любые.

## Прошивка

### Установка ESP-IDF:

1. Скачайте [ESP-IDF v5.5](https://docs.espressif.com/projects/esp-idf/en/v5.5/esp32s3/get-started/index.html)
2. Установите按照инструкции

### Прошивка:

```bash
# Клонируйте репозиторий
git clone https://github.com/Garrysoon/pico-fido.git
cd pico-fido
git submodule update --init --recursive

# Настройте окружение
idf.py set-target esp32s3

# Соберите
idf.py build

# Переведите устройство в режим загрузки (BOOT + USB)
# Прошейте
idf.py -p COM3 flash
```

## Устранение проблем

### Устройство не определяется:

1. Попробуйте **другой USB-кабель** (только зарядные не подходят!)
2. Попробуйте **другой USB-порт**
3. Проверьте **Диспетчер устройств** → есть ли "Yubikey 5A" или "USB-устройство"

### Windows не предлагает безопасный ключ:

1. Убедитесь что PIN установлен
2. **Настройки** → **Аккаунты** → **Варианты входа** → **Безопасный ключ** → **Управление**
3. Удалите старый ключ и добавьте заново

### Код OATH не работает:

1. Убедитесь что **время на компьютере синхронизировано**
2. Проверьте что выбранный тип (TOTP/HOTP) совпадает с настройками сервиса

### Кнопка не нажимается:

- При **регистрации** (MakeCredential) — кнопка **обязательна**
- При **входе** (GetAssertion) — кнопка **обязательна**
- При **настройке OATH** — кнопка **обязательна**

### Светодиод не горит:

- Проверьте настройки яркости (может быть 0)
- Проверьте правильный GPIO пин
- Проверьте тип LED (WS2812, Neopixel и т.д.)

## Часто задаваемые вопросы

**Q: Сколько аккаунтов может хранить ключ?**
A: Зависит от доступной памяти. Обычно 50-100 FIDO2 ключей и до 32 OATH аккаунтов.

**Q: Можно ли использовать несколько ключей одновременно?**
A: Да! Рекомендуется иметь 2 ключа: основной и backup.

**Q: Что если я забыл PIN?**
A: Можно сбросить PIN через **PicoKey App** или **TUI**, но все ключи будут удалены. Используйте backup!

**Q: Работает ли с Mac/Linux?**
A: Да! Устройство стандартное HID — работает везде.

**Q: Можно ли использовать для шифрования диска (BitLocker)?**
A: Да! Windows поддерживает BitLocker с FIDO2 ключами.
